The following conventions are used.

  • $J$,$K$ refers to the nuerons

  • there is no distinction between upper and lower indices, for example the elements of $w^{l,K}_J$ are the same as the elements of $w^l_{JK}$
  • partial derivatives are often notated as $\partial f/\partial x= \partial_x f$

What is a Neural Network?¶

From my perspecitve, a neural network is a non-linear statistical learning method. Neural networks are trained on data to make classification predictions. For example, a neural network might be used to predict whether a tumor is malignant or benign based on features of the tumor.

The figure below is a schematic diagram of a neural network (taken from wikipedia ). Each circle is a neuron. Each column of circles is called a layer. The first layer to the left is called the input layer. The number of neurons in the input layer is determined by the number of features in your training data. The ouput layer is the last layer on the right. The number of neurons in the ouput layer is determined by the number of classifications. The layers between the input and the output layers are called hidden. Each neuron recieves information from all neurons in the previous layer. This is indicated by the arrows in the figure. The information a single neuron recieves is a linear combination of the information of all neurons in the precedding layer. For example the information recieved in the top blue neuron is

\begin{align} W_1 \times \text{(output from 1st red neuron)} + W_2 \times \text{(output from 2nd red neuron)} + W_2 \text{(output from 3rd red neuron)} + \text{a bias term}~, \end{align}

where $W_1, W_2, W_3,$ and the bias are real numbers. The solution of a neural network is deteremind by finding out just exactly what all the $W$s and bias terms are.

Weights and Biases¶

Neurons are numbered top to bottom (my convention). Referring to the figure above, the neuron right below the word hidden has an index $(l,J)=(2,1)$, i.e., it is the first neuron in the second layer. The bottom neuron in the hidden layer has index $(l,J)=(2,4)$. The bottom neuron in the last layer has index $(l,J)=(3,2)$.

As mentioned previously, arrows in the figure (above) indicate that each neuron recieves information from all previous neurons. The information that the neuron recieves is a linear combination of the preceding layers. For example, the $(l,J)=(2,1)$ neuron in the figure above recieves information \begin{align} w^2_{11}X^1 + w^1_{12}X^2 + w^2_{13}X^3 + b^2_1 ~. \end{align} where the $w^l_{JK}$ are the weights, $b^l_J$ are the biases, and $X^J$ are the input features. A more compact notation is \begin{align} z^2_1=w^2_{J1}X^J +b^2_1~, \end{align} where the summation over the $J$ index is implied. In general the information at each neuron $(l,J)$ is \begin{align} z^l_J = w^{lK}_J a^{l-1}_K + b^l_J~, \end{align} where $a^l_J$ is the activated information passed from the previous layer to the next layer. (The notion of an activation will be explained in the next section.)

Activations¶

At each neuron an Activation functions is applied to the informaion $z^l_J = w^{lK}_J a^{l-1}_K + b^l_J$. An activation function does the following.

  • The simpilist activation function compares the output of the last layer to a threshold value. In the binary case, the output would be compared to a threshold value. If the output is higher than the threshold, one classificaiton is chosen, if it is less than the threshold, the other is chosen. This is essentially a step function. An activation function function like a hyperbolic tangent or sigmoid function smooths out the step function. In this way, small changes in input corespond to small changes in output.
  • An activation function makes the network nonlinear. If the output information from each layer was always of the form $z^l_J = w^{lK}_J a^{l-1}_K + b^l_J$, then the entire network would be linear.

Activation functions come in many forms. For a list of activation functions see here. In this notebook two activations will be utilized, the hyperbolic tangent and sigmoid functions. Below is a plot of the sigmoid function (taken from wikipedia). One can see that the activaiton function acts as a smoothing of the step function.

After each neuron recieves information from the preceding neurons an activation function is applied (sigmoid function used as an example) \begin{align} a^l_J=\sigma \left( w^l_{JK}X^K + b^l_J \right)~, \quad \sigma(z)= \frac{1}{1-\exp(-z)}~. \end{align} This information is then passed to every neuron in the next layer. The processes continues until the input information has passed through each layer. The solution to a neural network is determing the set of weights and bias such that the outputlayer gives as correct as possible predictions.

function z_l!(layer::Layer,dim)

  @inbounds for J in 1:dim[2]
    layer.z_l[J]=0.0

      layer.z_l[J]+=layer.w[K,J]*layer.a_lm[K]

Determining the Weights and Bias¶

A cost function quantifies how well a set of weights and biases predict an outcome. The simplist cost function is the sum of squares or the quadratic cost function \begin{align} C(w,b)= \frac{1}{N} \displaystyle \sum_{i,J} ( y_{i,J} - a^L_{i,J})^2~, \end{align} where $y_{i,J}$ is the known output and $a^L_{i,J}$ is the activation in the last layer. Solving for the wieghts and biases is an optimizaiton problem, one seeks to minimize the cost function. A typical optimization method is Gradient descent. The idea of gradient decsent is to move to the minimum (of $C$) iteratively from an initial guess by updating the weights and biases by (indices supressed) \begin{align} \begin{split} & w = w_0 - \eta \nabla_w C ~, \\ & b=b_0 - \eta \nabla_b C~, \end{split} \end{align} where $\eta$ is the learning rate parameter (much like a step size).

Mini-Batch Gradient Decsent¶

Mini-Batch gradient decsent is a means to speed up the computation by approximating the gradient with an average. The average is taken over a sample of randomly selected training data. The procedure is as follows.

  • randomly select a portion of the training data (called a mini-batch)
  • for each element in the mini-batch compute the gradients $\nabla_w C_i$ and $\nabla_b C_i$ (note this is the gradient of the cost function for a single training example)
  • average the computed gradients to approximate the true gradient, i.e., \begin{align} \begin{split} &\nabla_w C \approx \frac{1}{n_{batch} }\displaystyle \sum_i \nabla_w C_i~,\\ &\nabla_b C \approx \frac{1}{n_{batch} }\displaystyle \sum_i \nabla_b C_i~. \end{split} \end{align}

Backpropagation¶

The cost function is explictly a function of the activation in the last layer. But, the activation in the last layer is a function of all preciding activations. This makes calculating the gradient of the cost function difficult. Backpropagation is the solution. Backpropagation allows one to start at the last layer and then recursively move backwards through the network to calculate the gradient. The recursive aspect is what speeds the computation up. Micheal Nielson provides a great explanation of backpropagation in chapter 2 of his online book. The easiest explanation, although not entirely telling the whole story, envolves simple calculus. The chain rule gives the gradient in the last layer \begin{align} \begin{split} &\frac{\partial C}{\partial w^L}= \frac{\partial C}{\partial a^L}\frac{\partial \sigma^L}{\partial z^L}\frac{\partial z^L}{\partial w^L}~, \\ &\frac{\partial C}{\partial b^L}= \frac{\partial C}{\partial a^L}\frac{\partial \sigma^L}{\partial z^L}\frac{\partial z^L}{\partial b^L}~, \end{split} \end{align} where I have omitted the $J$ and $K$ indices for notational convience and defined $a^L=\sigma^L(z^L)$. Define the error in each layer as \begin{align} \Delta^l =\frac{\partial C}{\partial z^l}= \nabla_{a^l} C \odot \partial_{z^l} \sigma(z^l)~. \end{align} The change in the error with respect to different layers is given by the chain rule \begin{align} \begin{split} \Delta^l_J & =\frac{\partial C}{\partial z^{l+1}_J} \frac{\partial z^{l+1}_K}{\partial z^l_J} \\ & = \Delta^{l+1} \frac{\partial z^{l+1}_K}{\partial z^l_J}~. \end{split} \end{align} Recall the definition of the activation $a^l_J=\sigma(z^l_J)$ and the intermediate information $z^{l+1}_J=w^{l+1,K}_J a^{l}_K + b^{l+1}_J$. Then, \begin{align} \frac{ \partial z^{l+1}_J}{\partial z^l_K}= w^{l+1,K}_J \partial_{z^l_K} \sigma(z^l_K)~. \end{align} Combining the two previous equations gives \begin{align} \Delta^l_J= \Delta^{l+1}_J w^{l+1, K}_J \partial_{z^l_K} \sigma(z^l_K)~. \end{align} This is a recursive relation for the error $\Delta^l$. The gradients for each layer are \begin{align} \begin{split} & \frac{\partial C}{\partial w^l}= \frac{\partial C}{\partial z^l} \frac{\partial z^l}{\partial w^l} = \Delta^l a^{l-1} ~,\\ & \frac{\partial C}{\partial b^l}= \frac{\partial C}{\partial z^l} \frac{\partial z^l}{\partial b^l} = \Delta^l ~. \end{split} \end{align}

Code part II¶

step 1. set the initial weights and biases ($w^l_{JK}~ \&~ b^l_J$) by randomly drawing from a Gaussian distribution

step 2. start the mini-batch gradient descent

  • begin a loop through each training epoch
  • create mini-batches of a predetermined size by randomly sampeling the training data
  • loop through each element (i.e. a single training data ) of the mini-batch data

step 3. for each training example compute the gradients of the cost function ($\nabla_w C~ \& \nabla_b C$)

step 3.1 for a training sample $x_J$ move forward through the network

  • $a^1_J=x_J$, set the initial activation to $x=\text{the training data features}$
  • calculate the intermediate infromation $z^l_J$ and activation $a^l_J$ for each layer
  • $z^l_J= w^{l,K}_J a^{l-1}_J + b^l_J $
  • $a^l_J=\sigma(z^l_J)$.

step 3.2 apply backward propagation

  • In the last layer compute $\Delta^L=\nabla_{a^L} C \odot \partial_{z^L} \sigma(z^L)$
  • start a loop propagating from the last layer to the second layer
  • $\Delta^l_J= \Delta^{l+1}_J w^{l+1, K}_J \partial_{z^l_K} \sigma(z^l_K)$
  • $\partial_{b^{l_J}}C_i= \Delta^l_J $
  • $\partial_{w^l_{JK}}C_i=a^{l-1}_K \Delta^l_J$
  • note that the $i$ indicates the cost is computed for a single training example

step 4 compute $\nabla_w C$ and $\nabla_b C$ by averaging and then update $w$ and $b$

  • $w^l_{JK}=w^l_{JK} - \eta \frac{1}{n_{batch}} \displaystyle \sum_{i}\partial_{b^{l_J}}C_i$
  • $b^l_J= b^l_J - \eta\frac{1}{n_{batch}} \displaystyle \sum_{i}\partial_{w^l_{JK}}C_i$
  • ($n_{batch}$ is the number of elements in a mini-batch)

step 5 make predictions by applying the forward alogrithm with the weights and biases determined from the stochastic gradient decsent

  • select a classficication by choosing the neurons that have the largest activation in the output layer
  • For example, if you have three classfiers numerically identified as 1,2,3, you have three output neurons. Suppose, the second neuron has the largest activation. This network predicts that the output is the second classification (2).

function backprop!(NN,x_in,y,dC_da,dcda)

    L=size(NN.Layers,1)

    # moveforward through the network
    forward_pass!(NN,x_in)

    dC_da(dcda,NN.Layers[L].a_l,y)
    set_Delta_l!(NN.Layers[L].Delta_l,dcda,NN.Layers[L].da_dz)

    @inbounds for l in L:-1:2     
           axpy!(1,NN.Layers[l].Delta_l,NN.Layers[l].C_b)  
           update_C_w!(NN.Layers[l].C_w,NN.Layers[l-1].a_l,NN.Layers[l].Delta_l)
           update_Delta_l!(NN.Layers[l-1].Delta_l,NN.Layers[l].Delta_l,NN.Layers[l].w,NN.Layers[l-1].da_dz)
    end
    # manually do last layer (actually the first layer)
    axpy!(1,NN.Layers[1].Delta_l,NN.Layers[1].C_b)
    update_C_w!(NN.Layers[1].C_w,x_in,NN.Layers[1].Delta_l)
end

            predict_Y!(output,NN,train_data.X,train_data.Y_vector,test_data.X,test_data.Y_vector,0)

        else
            # should be stored somewhere as an accessible type          
            output=results{Float64,Int64}(epochs,n_targets,n_samples,1)
            predict_Y!(output,NN, train_data.X,train_data.Y_vector,0)
        
        end
    end;
    
#     #= Combine data to one array
#     so that can shuffle only one array.
#     Also, take transpose so that arrays can 
#     be searched by columns. Apparently column 
#     access is faster than row in Julia. (Not sure 
#     if that helps, but I found it to be just so slightly
#     faster) =#
    
    const data=hcat(train_data.X,train_data.Y_onehot)'
    n_tot=size(data,1)
    data_rand=zeros(data)
    id=collect(1:size(data,2))

    
    local start=0
    local fin=0

    #Delta_l=[zeros(b) for b in NN.bias] # want to put Delta_l in a data type, costing allocations
    zero_C_theta!(NN)
        
     for i_epoch in 1:epochs
        
        # randomly shuffle the data 
        #data[:,shuffle(1:end)]
        shuffle_dim2!(data_rand,data,id)
        
        local n_batch=1
        for i in 1:n_samples

            # employ backpropagation
            backprop!(NN,data_rand[1:n_features,i],data_rand[n_features+1:n_tot,i],dC_da,dcda)
            
            
            if n_batch==mini_size
                # update weights and bias
                standard_update!(NN,eta/float(n_batch))
                # reinitialize cost function and n_batch
                zero_C_theta!(NN)
                n_batch=0
                
            elseif i ==n_samples
                standard_update!(NN,eta/float(n_batch))
                # reinitialize cost function and n_batch
                zero_C_theta!(NN)
            end 
           n_batch+=1 
        end
        if track
            if yes_test
                predict_Y!(output,NN,train_data.X,train_data.Y_vector,test_data.X,test_data.Y_vector,i_epoch)
            else
                predict_Y!(output,NN, train_data.X,train_data.Y_vector,i_epoch)
            end
        end
    end
    if track
        return output
    end
end;

MNIST Analysis¶

The MNIST data set is set of images with $28 \times 28$ pixels. The analysis will consider each pixel as a feature. Consequently each image needs to be converted into an array of length 784. The function convert_mnist does this.

The MNIST analysis will be done on a neural network with three layers, the final two being sigmoid layers. The other parameters for the neural network are: the learning rate $\eta=3.0$, number of training epochs is 20, and the mini-batch size will be 20. To quantify the progress of learning the fraction of accurately predicted training data and testing data will tracked for each epoch. The plot below shows this result.

function convert_mnist(X)
    # functions takes the matrix of 
    # pixels and converts into a single 
    # array 
    n=size(X,3)
    X_out=zeros(n,784)
    for i in 1:n
        sample=X[:,:,i]
        sample=reshape(sample,1,784)
        X_out[i,:]=sample
    end
    return X_out
end
